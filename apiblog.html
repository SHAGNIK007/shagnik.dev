Title: ShaggyAI




To try it out click here => ShaggyAI

Why I Built This

I built this project mainly to understand how APIs actually work in real-world applications. I also wanted some hands-on experience building and working with Flask APIs instead of just reading about them.

The goal was simple: build something practical while learning how backend services interact with AI APIs.


The First Approach

When I first started the project, I used the Gemini API to handle the AI functionality.

Everything worked fine during development. But when I tried deploying the project on Vercel, things started breaking.

The deployment kept failing because Vercel couldn't properly recognize or support the Gemini API version I was using. This caused repeated errors during the build process, and the project simply wouldn't deploy.


Finding Another Solution

After running into this problem multiple times, I started looking for alternatives.

That's when I discovered the Groq API. It’s designed for fast AI inference and works well with open-source models (often used together with tools like Ollama).

So I decided to refactor parts of the project and replace the Gemini API integration with Groq.


What Happened After Switching

Switching to Groq actually ended up being a really good decision.

A few things immediately improved:

  • • The deployment on Vercel started working without errors
  • • The API responses were noticeably faster
  • • Groq is free to use and more open-source friendly

After updating the implementation, the project deployed successfully and ran much more smoothly.


P.S => Also its just api integration so the name ShaggyAI is just for fun :>
